# gtop

Lightweight macOS live system monitor in a compact floating HUD.

## Overview

`gtop` is a menu bar utility for macOS that shows live system status in a small, fast HUD.

Current focus:

- native macOS utility feel
- compact floating monitor window
- low-overhead local system monitoring
- simple control from the menu bar

## Features

- Menu bar app
- Left-click the menu bar icon to show or hide the HUD
- Right-click the menu bar icon to:
  - open keyboard shortcut settings
  - toggle foreground mode
  - toggle launch at login
  - quit the app
- HUD resource cards:
  - CPU
  - Memory
  - Disk
  - Network
- Click a card to view top processes for that resource
- If foreground mode is off, clicking outside the HUD hides it
- Custom app icon via `Assets.xcassets`

## Tech Stack

- Swift
- SwiftUI
- AppKit bridge
- Xcode project generated by a Ruby script

## Repository Layout

- `gtop/`: app source
- `gtopTests/`: unit tests
- `gtop.xcodeproj/`: generated Xcode project
- `tools/generate_xcodeproj.rb`: project generator
- `tools/generate_app_icon.swift`: app icon asset generator helper

## Requirements

- macOS 14+
- Xcode
- `swiftlint`
- Ruby with the `xcodeproj` gem if you want to regenerate the Xcode project

## Open In Xcode

```bash
open gtop.xcodeproj
```

## Build

```bash
xcodebuild -project gtop.xcodeproj -scheme gtop build
```

## Test

```bash
xcodebuild -project gtop.xcodeproj -scheme gtop test
```

## Lint

```bash
swiftlint lint --strict .
```

## Regenerate The Xcode Project

```bash
ruby tools/generate_xcodeproj.rb
```

## Launch At Login Note

`gtop` includes a launch-at-login toggle using `ServiceManagement`.

In development builds, macOS may reject registration depending on signing and app location. The UI flow is implemented, but successful registration is most reliable with a properly signed app build.

## Notes

- Local-only utility, no telemetry
- The current build may show an asset catalog warning for missing `AccentColor`; this does not block the app
- The HUD is intentionally compact and utility-focused rather than highly configurable
